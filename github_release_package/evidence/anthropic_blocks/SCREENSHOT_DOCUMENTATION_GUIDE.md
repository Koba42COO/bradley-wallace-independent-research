# Anthropic Anticompetitive Behavior - Screenshot Documentation Guide

## How to Document Screenshots

### File Naming Convention

```
anthropic_block_incident_[NUMBER]_[DATE]_[DESCRIPTION].png

Examples:
- anthropic_block_incident_01_2025-11-16_policy_investigation.png
- anthropic_block_incident_02_2025-11-16_conversation_review.png
- anthropic_block_incident_03_2025-11-16_ip_theft_continuation.png
```

### Required Information for Each Screenshot

1. **Incident Number:** Sequential number (01, 02, 03, etc.)
2. **Date:** YYYY-MM-DD format
3. **Description:** Brief description of what triggered the block
4. **Timestamp:** Exact time when block occurred
5. **Context:** What user was trying to investigate

### Screenshot Checklist

For each blocking incident, capture:

- [ ] Full conversation window showing the block
- [ ] Error message displayed
- [ ] User's request that triggered the block
- [ ] System's response before blocking
- [ ] Timestamp visible in screenshot
- [ ] Browser/application information
- [ ] Any relevant context or previous messages

### Documentation Template

For each screenshot, create a corresponding markdown file:

```markdown
# Incident [NUMBER]: [DESCRIPTION]

**Date:** [YYYY-MM-DD]  
**Time:** [HH:MM:SS]  
**Trigger:** [What user said/asked]  
**Response:** [System's response before block]  
**Block Message:** [Exact error message]  

## Context
[Describe what user was investigating]

## Evidence
- Screenshot: `anthropic_block_incident_[NUMBER]_[DATE]_[DESCRIPTION].png`
- Conversation log: [If available]
- Pattern: [How this fits the pattern]

## Legal Implication
[How this demonstrates anticompetitive behavior]
```
